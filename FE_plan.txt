Nihongo Flashcard App вЂ” Frontend Architecture & Implementation Plan
================================================================================

1. TECH STACK
================================================================================

CORE FRAMEWORK:
- React 18+ (with Hooks)
- TypeScript (for type safety)
- Vite (build tool, faster than CRA)

ROUTING & STATE:
- React Router v6 (client-side routing)
- Zustand / Redux Toolkit (global state management)
- React Query (TanStack Query) - server state, caching, API calls
- Jotai / Recoil (atomic state - optional for complex states)

STYLING:
- Tailwind CSS 3+ (utility-first CSS)
- Headless UI / Radix UI (accessible component primitives)
- Framer Motion (animations)
- clsx / classnames (conditional classes)

UI COMPONENTS:
- Custom components built on Radix UI
- shadcn/ui (copy-paste component library)
- Lucide React (icon library - Japanese-friendly)
- react-icons (supplementary icons)

FORMS & VALIDATION:
- React Hook Form (performant form handling)
- Zod (schema validation)
- @hookform/resolvers (connect Zod with React Hook Form)

API & DATA FETCHING:
- Axios (HTTP client)
- React Query (data fetching, caching, synchronization)
- Socket.io-client (real-time features - optional for future chat/notifications)

AUTHENTICATION:
- JWT tokens (stored in httpOnly cookies + localStorage fallback)
- axios interceptors (auto-attach token, refresh token logic)
- Protected Route wrapper component

FILE HANDLING:
- react-dropzone (drag & drop PDF upload)
- pdf.js / react-pdf (PDF preview)
- file-saver (download flashcards as JSON/CSV)

RICH TEXT EDITOR (for Blog):
- TipTap / Lexical (modern WYSIWYG editor)
- DOMPurify (sanitize HTML content)

FLASHCARD FEATURES:
- react-card-flip (flip animation)
- react-swipeable-views (swipe gestures for mobile)
- use-sound (audio feedback for correct/wrong answers)

CHARTS & ANALYTICS:
- Recharts / Chart.js (study progress visualization)
- react-chartjs-2 (wrapper for Chart.js)

UTILITIES:
- date-fns (date manipulation - lighter than moment.js)
- lodash-es (utility functions, tree-shakeable)
- react-hot-toast / sonner (toast notifications)
- react-helmet-async (SEO meta tags)

TESTING:
- Vitest (unit testing, Vite-compatible)
- React Testing Library (component testing)
- Playwright / Cypress (E2E testing)

CODE QUALITY:
- ESLint + Prettier (code formatting)
- Husky + lint-staged (pre-commit hooks)
- TypeScript strict mode

DEPLOYMENT:
- Vercel / Netlify (static hosting)
- Docker (containerization - optional)
- GitHub Actions (CI/CD)

4. KEY FEATURES IMPLEMENTATION
================================================================================

A. AUTHENTICATION FLOW
-----------------------
1. Login/Register:
   - Form validation with Zod
   - POST to /api/auth/login or /api/auth/register
   - Store JWT token in localStorage + httpOnly cookie
   - Save user info to Zustand authStore
   - Redirect to /dashboard

2. Google OAuth:
   - Use react-oauth/google package
   - Backend returns JWT after Google verification
   - Same flow as email/password login

3. Protected Routes:
   - ProtectedRoute wrapper checks authStore.isAuthenticated
   - Redirect to /login if not authenticated
   - Axios interceptor auto-attaches Authorization header

4. Token Refresh:
   - Axios response interceptor catches 401
   - Call /api/auth/refresh-token
   - Retry failed request with new token


B. FREEMIUM RATE LIMITING (Frontend)
-------------------------------------
1. Before calling /api/translate or /api/upload:
   - Check authStore.user.isPro and user.proExpiresAt
   - If isPro && proExpiresAt > now в†’ Allow action
   - If not PRO в†’ Show usage badge (e.g., "3/10 translations today")

2. On 403 RateLimitExceeded from backend:
   - Show modal: "Daily limit reached! Upgrade to PRO for unlimited access"
   - Display pricing cards with PayOS payment link

3. Usage Badge Component:
   - Fetch /api/users/daily-usage on mount
   - Display: "AI Translations: 7/10 | PDF Uploads: 1/2"
   - Real-time update after each action
